Introducción

Este documento pretende guiar de manera clara y estructurada el proceso de

configuración del sistema Jeeve, proporcionando las instrucciones necesarias para su

correcta implementación y funcionamiento.

A lo largo de este documento, se describen los pasos, parámetros y consideraciones

clave que deben tenerse en cuenta para garantizar una configuración adecuada,

alineada con las necesidades operativas de la organización.

Como toda herramienta tecnológica el resultado satisfactorio de la adopción dependerá

en gran parte del entendimiento que los usuarios tengan acerca del uso de la plataforma,

logrando con esto tener claridad en los procesos y los alcances de la propia herramienta.

Recomendamos leer muy detenidamente cada paso y al tiempo ir verificando las

funcionalidades que se describen en este instructivo.

¿Qué es Jeeves?

Jeeves es una plataforma tecnológica utilizada por las empresas para la gestión y

administración de tarjetas de crédito virtuales corporativas. Esta solución permite emitir,

controlar y monitorear tarjetas virtuales que son utilizadas para realizar pagos asociados a

procesos empresariales, como reservas de viajes, gastos corporativos u otras transacciones

autorizadas por la organización.

La plataforma ofrece servicios que permiten a otros sistemas empresariales integrarse

con Jeeves mediante APIs o servicios web, con el fin de automatizar la creación, consulta,

administración y cancelación de tarjetas virtuales.

¿Qué hace Jeeves dentro del proceso?

Dentro de la arquitectura del sistema, Jeeves cumple el rol de proveedor del servicio

financiero y de emisión de tarjetas virtuales. A través de la integración implementada, el

sistema interno puede comunicarse con la plataforma para ejecutar diferentes

operaciones, entre ellas:

• Crear tarjetas de crédito virtuales, asociadas a una reserva o transacción

específica.

• Eliminar o cancelar tarjetas virtuales cuando ya no se requieren o cuando una

reserva es anulada.

• Gestionar información relacionada con los medios de pago utilizados en los

procesos corporativos.

Esto permite que los procesos empresariales que requieren un medio de pago puedan

ejecutarse de forma automática, segura y controlada, sin necesidad de manejar tarjetas

físicas o procesos.

Beneficios de u tilizar Jeeves

El uso de la plataforma Jeeves dentro de la integración proporciona varios beneficios

para la organización:

• Automatización de procesos de pago dentro de los sistemas corporativos.

• Mayor seguridad, ya que las tarjetas virtuales pueden generarse con límites y

condiciones específicas.

• Reducción de riesgos asociados al uso de tarjetas.

Como usar Jeeves dentro de la plataforma Kontroltravel / xnet?

En el gestor, se debe de crear la conexión de jeeves y asociársela a la entidad que va a

usar como pasarela de pago tarjetas virtuales

Configuración de los parámetros de conexión

La configuración de los parámetros de conexión contiene la información necesaria para

establecer la comunicación entre el sistema y la API de Jeeves. Algunos de estos parámetros

son valores fijos definidos por la plataforma Jeeves, mientras que otros son valores variables que el

proveedor entrega a cada agencia para permitir la autenticación y el acceso a los servicios.

Definición de los parámetros

A continuación, se describen los parámetros utilizados para la configuración de la integración:

• grant_type

Valor utilizado para el proceso de autenticación con la API. Este valor es fijo y siempre

debe configurarse como:

client_credentials

• scope

Define los permisos que se solicitan para acceder a los servicios de la API de Jeeves.

Este valor es fijo y corresponde a:

cards:read cards:write

• endPoint_autenticacion

Endpoint utilizado para realizar el proceso de autenticación y obtener el token de

acceso.

/oauth/token

• endPoint_DarAltaTC

Endpoint utilizado para realizar la creación de tarjetas de crédito virtuales en la

plataforma Jeeves.

/v1/cards

• url_ws

Corresponde a la URL base del servicio web de Jeeves.

Actualmente, en la integración se encuentra configurado el ambiente de pruebas (test)

proporcionado por el proveedor.

• client_secret

Clave de autenticación proporcionada por Jeeves a cada agencia.

Este valor es único por agencia y se utiliza junto con el client_id para el proceso

de autenticación en la API.

• client_id

Identificador o usuario proporcionado por Jeeves para cada agencia.

Este valor se utiliza en conjunto con el client_secret para obtener el token de

acceso

Parámetros asociados a la configuración del certificado

Existen parámetros adicionales que dependen de la configuración definida en Ideas

Fractal para el manejo de certificados digitales utilizados en la integración.

UsaCertificado

Define el mecanismo mediante el cual se utilizará el certificado digital durante la

integración.

Valores posibles:

• local: la integración utilizará el certificado instalado directamente en el servidor

donde se ejecuta el sistema.

• AWS: la integración utilizará el certificado almacenado y gestionado a través de

servicios de Amazon Web Services (AWS).

CertificadoAWS

Este valor corresponde al identificador del certificado configurado en Amazon AWS.

Es proporcionado por el área de infraestructura, según la configuración realizada en la

plataforma.

Certificado

En este campo se debe registrar el Thumbprint del certificado digital instalado en el

servidor.

Este valor también es entregado por el área de infraestructura, una vez el certificado ha

sido configurado correctamente en el servidor.

Nota importante

Si el certificado utilizado es el mismo para todas las agencias, los valores correspondientes a:

• UsaCertificado

• CertificadoAWS

• Certificado

serán los mismos para todas las configuraciones de agencia.

En estos casos, únicamente deberán variar los siguientes parámetros para cada agencia:

• client_id

• client_secret

Estos valores son únicos por agencia y son proporcionados directamente por el proveedor

Jeeves, y estas credenciales se guardarán en la bóveda de ideas fractal para mayor

seguridad de la agencia.

Las agencias tendrán disponible la creación de tarjetas de crédito y poder hacer uso en el

portal como tarjetas de crédito convencionales.

Opción para crear una tarjeta de crédito virtual en jeeves

Definición de campos para la creación de la tarjeta virtual

A continuación, se describen los campos requeridos para el proceso de creación de una

tarjeta de crédito virtual en la plataforma Jeeves.

• Fecha inicio de uso de la tarjeta

Corresponde a la fecha a partir de la cual la tarjeta virtual creada en Jeeves comienza

a estar habilitada para su uso.

• Fecha final de uso de la tarjeta

Indica la fecha límite de vigencia de la tarjeta virtual, es decir, hasta cuándo podrá ser

utilizada para realizar transacciones.

• Cantidad de transacciones

Este parámetro define el número máximo de veces que la tarjeta virtual podrá ser

utilizada para realizar transacciones.

El valor configurado establece el límite de usos permitidos para la tarjeta, lo cual

permite controlar y restringir la cantidad de operaciones que pueden realizarse con

ella. Una vez alcanzado este límite, la tarjeta no podrá seguir siendo utilizada para

nuevas transacciones.

• Monto total solicitado en la tarjeta de crédito

Este parámetro corresponde al valor total de crédito que se asignará a la tarjeta virtual

al momento de su creación. El monto definido representa el límite máximo disponible

para realizar transacciones con la tarjeta. Todas las operaciones realizadas con la tarjeta

virtual se descontarán de este valor hasta alcanzar el límite establecido.

• Entidad

Corresponde a la entidad o agencia asociada a la tarjeta de crédito virtual dentro del

sistema.

• Nombre de referencia para la creación de la tarjeta (TC)

Este campo permite definir el nombre de la persona o referencia asociada a la

creación de la tarjeta virtual.

Es importante tener en cuenta que no se permite ingresar más de 200 caracteres, ya que

el sistema de Jeeves rechazará la solicitud si se supera este límite.

• Dirección de residencia

Información correspondiente a la dirección de residencia asociada al titular o

referencia de la tarjeta virtual.

• Teléfono de contacto

Número telefónico de contacto asociado al titular o referencia de la tarjeta virtual,

el cual podrá ser utilizado en caso de requerirse validaciones o información adicional

relacionada con la tarjeta.

Opcion eliminar tarjeta virtual

Esta opción se presenta como una lista desplegable, en la cual se mostrarán todas las

tarjetas virtuales asociadas a Jeeves que se encuentran actualmente activas en el sistema.

En el listado se visualizará la siguiente información de cada tarjeta:

• Nombre o referencia configurada de la tarjeta

• Monto asignado a la tarjeta

• Los últimos cuatro dígitos de la tarjeta, correspondientes al número generado por

Jeeves

Esta información permite identificar fácilmente cada tarjeta antes de realizar alguna acción

sobre ella.

Una vez seleccionada la tarjeta deseada y al hacer clic en el botón "Ejecutar", el sistema

procederá a:

1. Enviar la solicitud de eliminación de la tarjeta a la plataforma Jeeves.

2. Actualizar el estado de la tarjeta en el sistema KontrolTravel/XNET, marcándola como

inactiva.

De esta forma se garantiza que la tarjeta no pueda volver a ser utilizada ni en Jeeves ni en el sistema

interno.

Uso de tarjetas virtuales para pago de reservas

Para realizar pagos de reservas utilizando tarjetas virtuales integradas con Jeeves,

es necesario que la entidad tenga configurada esta opción dentro del formulario de

Opciones de Pago.

Emisión manual

En los casos donde la emisión de la reserva se realiza de forma manual, la entidad debe

tener habilitada la opción de tarjetas virtuales en las Opciones de Pago.

Al momento de realizar el pago de la reserva, el sistema creará internamente una tarjeta de

crédito virtual en la plataforma Jeeves, utilizando el monto total de la reserva como valor

disponible en la tarjeta. Esta tarjeta será utilizada como medio de pago para completar la

emisión de la reserva.

Emisión automática

Para los procesos de emisión automática, es necesario configurar el siguiente parámetro

en el sistema:

• Formapagoclientecorporativo = V

Cuando este parámetro se encuentra configurado, el sistema creará automáticamente las

tarjetas virtuales a través de la integración con Jeeves, permitiendo realizar el pago y completar

la emisión de la reserva sin intervención manual.

En este proceso, la tarjeta virtual se genera utilizando la información definida en el OBT y el

monto correspondiente a la reserva.

De esta manera, la tarjeta virtual será creada automáticamente y utilizada como medio de

pago para completar la transacción de la reserva, garantizando un proceso seguro y automatizado.
