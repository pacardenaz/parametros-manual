Introducción

El módulo de Ventana de Precios Más Económicos tiene como objetivo ayudar al usuario

a identificar opciones de vuelo más económicas en relación con el vuelo que ha

seleccionado, sin perder de vista las condiciones tarifarias que este eligió originalmente.

Este módulo busca mejorar la experiencia de compra, permitiendo:

• Comparar precios de manera clara y transparente.

• Visualizar posibles ahorros.

• Mantener coherencia entre las condiciones de las tarifas comparadas.

• Facilitar la toma de decisiones del usuario final.

La funcionalidad puede adaptarse a distintas necesidades operativas mediante

parámetros de configuración, lo que permite a cada agencia definir cómo, cuándo y bajo

qué criterios se muestran las opciones más económicas.

Fundamentos

1. Principio de comparación justa

No todas las tarifas aéreas ofrecen las mismas condiciones. Algunas incluyen equipaje en

bodega, otras solo morral; algunas permiten cambios, otras no.

Por este motivo, el sistema está diseñado para:

• Evitar comparaciones incorrectas entre tarifas con condiciones muy diferentes.

• Garantizar que los vuelos sugeridos como “más económicos” sean comparables

con el vuelo seleccionado.

Este principio se implementa mediante la agrupación de familias tarifarias, donde se

comparan únicamente tarifas con condiciones similares.

2. Familias tarifarias y su variabilidad

Las aerolíneas definen sus propias familias tarifarias (por ejemplo: LIGHT, BASIC, FULL,

PREMIUM). Sin embargo:

• Los nombres de las familias no son estándar.

• Un mismo tipo de tarifa puede tener nombres distintos según:

o La aerolínea.

o El proveedor de información (Amadeus, Sabre, NDC directo, etc.).

Por esta razón, el sistema permite que cada agencia configure sus propios grupos de

familias, alineándolos según criterios comunes como:

• Cantidad de equipaje.

• Posibilidad de cambios.

• Nivel de flexibilidad.

3. Enfoque en la experiencia del usuario

El módulo está diseñado para mostrar solo la información necesaria en cada escenario:

• En su versión simple, presenta alternativas económicas de forma directa.

• En su versión avanzada, ofrece:

• Tomar decisiones informadas.

• Reducir el tiempo de búsqueda de alternativas.

4. Flexibilidad mediante configuración

El comportamiento del módulo se controla completamente a través de parámetros, lo

que permite:

• Activar o desactivar la funcionalidad.

• Definir cuántas opciones económicas mostrar.

• Establecer reglas de comparación entre tarifas.

Gracias a esta flexibilidad, el módulo puede adaptarse a:

• Políticas comerciales de la agencia.

• Diferentes tipos de mercado (principalmente vuelos nacionales).

• Preferencias del cliente final.

5. Alcance funcional

Este flujo está pensado principalmente para:

• Vuelos nacionales, donde la cantidad de familias tarifarias es más controlada.

En vuelos internacionales:

• La funcionalidad puede operar.

Objetivo del módulo

Este módulo permite informar y sugerir al usuario vuelos más económicos en

comparación con el vuelo que ha seleccionado, mostrando una ventana con opciones

alternativas según diferentes criterios de configuración.

El sistema cuenta con:

• Una ventana simple (modo anterior).

• Una ventana nueva y personalizable, con mayor nivel de detalle y posibilidad de

selección directa.

Parámetros de configuración

Parámetro: VERIFICASVUELOSMASECONOMICOS

• Descripción: Controla si se muestra o no la ventana de precios más económicos y

qué tipo de ventana se utiliza.

• Valores soportados:

Valor Comportamiento

Activa la ventana simple (funcionamiento anterior). Muestra de

S

forma básica los vuelos más económicos.

Activa la nueva ventana avanzada con más información y opción de

C

selección directa del vuelo.

N o X Desactiva completamente la ventana de precios económicos.

Ejemplos

• VERIFICASVUELOSMASECONOMICOS = S → Muestra la ventana simple.

• VERIFICASVUELOSMASECONOMICOS = C → Muestra la ventana avanzada.

• VERIFICASVUELOSMASECONOMICOS = N → No se muestra ventana.

Parámetro: LIMITEOPCIONESPRECIOSBAJOS

• Descripción: Define la cantidad máxima de opciones de vuelos económicos que se

mostrarán en la ventana.

• Valores soportados:

Valor Comportamiento

2 Muestra los dos vuelos más económicos.

N Muestra hasta N opciones económicas.

Parámetro: COMBINACIONFAMILIESVENTANAPRECIOSECONOMICOS

• Descripción: Permite agrupar familias tarifarias para que la comparación de

precios se realice solo entre tarifas equivalentes, aunque tengan nombres

distintos según aerolínea o proveedor (source).

Este parámetro es opcional, pero altamente recomendado para obtener

comparaciones correctas.

o Los nombres varían entre aerolíneas.

o Los nombres pueden cambiar según el source (Amadeus, Sabre, NDC)

Por ello, este parámetro permite:

o Agrupar familias con condiciones similares (maletas, cambios,

reembolsos).

o Comparar solo vuelos equivalentes entre sí.

Soluciones de viajes

• Al tratarse de un parámetro de valor abierto, es necesario que su configuración no

contenga espacios ni saltos de línea. Aunque en los ejemplos se presenta el valor

dividido en varias líneas con fines ilustrativos, al momento de configurar el

parámetro en una entidad este debe ingresarse como un único texto continuo, sin

espacios intermedios ni saltos de línea, para garantizar su correcto

funcionamiento.

Parámetro:TOLERANCIAHORASPRECIOBAJONAL / TOLERANCIAHORASPRECIOBAJOINTER

• Descripción: Estos parámetros controlan la tolerancia horaria utilizada para

evaluar y sugerir vuelos más económicos, tomando como referencia la hora del

vuelo seleccionado por el usuario.

La lógica de funcionamiento es la misma para ambos parámetros; la única

diferencia es el tipo de itinerario al que aplican:

o TOLERANCIAHORASPRECIOBAJONAL: aplica a vuelos nacionales.

o TOLERANCIAHORASPRECIOBAJOINTER: aplica a vuelos internacionales.

Cada parámetro puede configurarse con un valor diferente, permitiendo definir

rangos horarios distintos según el tipo de ruta. Estos parámetros reemplazan el

comportamiento del parámetro obsoleto TOLERANCIAHORASPRECIOBAJO.

• Valores soportados:

Valor Comportamiento

Define el rango horario en horas para la búsqueda de vuelos más

1 a 24 económicos. El valor se aplica antes y después de la hora del vuelo

seleccionado.

Formato

• Valor entero.

• Sin minutos.

• No se permiten decimales.

Preguntas Frecuentes

• ¿Qué ocurre si no se configuran las familias tarifarias?

El sistema comparará el vuelo seleccionado contra el vuelo más económico

disponible, sin validar equivalencia de condiciones, lo que puede generar

comparaciones con tarifas de menor nivel.

• ¿La ventana de precios más económicos es obligatoria?

No. La funcionalidad puede activarse o desactivarse mediante el parámetro

Verificasvuelosmaseconomicos.

• ¿La tolerancia horaria aplica siempre?

Solo aplica si los parámetros TOLERANCIAHORASPRECIOBAJONAL o

TOLERANCIAHORASPRECIOBAJOINTER están configurados. De lo contrario, no se

restringe el rango horario.

• ¿Puedo usar valores distintos de tolerancia horaria para vuelos nacionales e

internacionales?

Sí. Ambos parámetros funcionan de la misma forma, pero permiten definir rangos

horarios diferentes según el tipo de itinerario.
