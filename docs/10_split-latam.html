Introducción.

Con el objetivo de optimizar la gestión de reservas y brindar mayor flexibilidad operativa a las agencias de viaje, se pone a disposición la funcionalidad de SPLIT de pasajeros para reservas LATAM vía NDC.

Esta funcionalidad permite dividir pasajeros pertenecientes a una reserva ya emitida, generando los registros, órdenes y documentos correspondientes para cada nueva reserva resultante, siempre en cumplimiento de las políticas y restricciones definidas por el proveedor.

El presente documento tiene como finalidad describir el alcance de la funcionalidad de SPLIT, así como detallar las consideraciones operativas que deben evaluarse previo a su utilización y los permisos de usuario requeridos para acceder y ejecutar este proceso correctamente.

Funcionalidad SPLIT

La funcionalidad SPLIT LATAM permite dividir uno o más pasajeros de una reserva ya emitida, generando una nueva orden y un nuevo localizador interno (LOC) en Kontrol, asociado al registro creado a partir del SPLIT realizado.

Una vez emitida la reserva, se podrá realizar este Split seleccionando la opción “Dividir PNR”.

Figura 1

El sistema desplegara una nueva ventana en la que se visualizarán los nombres de todos los pasajeros asociados a la reserva. Desde esta ventana, la agencia deberá seleccionar los pasajeros que desea dividir de la reserva original para continuar con el proceso.

Consideraciones importantes antes de realizar un SPLIT

El SPLIT no es reversible

Una vez realizada la división, no es posible anular la nueva orden generada desde el sistema. Si la agencia requiere la anulación de los tiquetes divididos (nueva orden), deberá gestionarse directamente con LATAM.

Consideraciones operativas

Teniendo en cuenta el anterior punto, en escenarios donde una reserva inicial contiene uno o más pasajeros, y posteriormente se identifica que solo alguno de ellos realizara el viaje, mientras que otros deben ser cancelados se recomienda aplicar el siguiente procedimiento:

Ejecutar el Split (nueva orden) de los pasajeros que, si van a realizar el viaje, de manera que estos queden asociados a nuevos localizadores y a nuevas reservas.

Mantener en la reserva original a los pasajeros que no realizarán el viaje, con el fin de que su cancelación pueda efectuarse directamente desde la herramienta, sin necesidad de gestionar el proceso con LATAM.

De esta forma, los pasajeros que no viajan pueden ser cancelados directamente en la plataforma, mientras que los pasajeros que sí realizarán el viaje continúan su proceso normal en las reservas generadas

Generación de nueva orden y localizador.

Al dividir una reserva, Latam genera una nueva orden, y de igual forma, el sistema genera un nuevo localizador interno (LOC) para los pasajeros divididos.

Registro en el histórico del sistema

La información del SPLIT queda registrada en el histórico de Kontrol permitiendo a la agencia

consultar al detalle el proceso cuando lo requiera.

Figura 3

SPLIT de pasajeros menores

En caso de realizar un SPLIT de un niño, este siempre deberá ir acompañado de un adulto. No será posible dividir un menor de edad de forma individual.

Cambios y reemisiones posteriores al SPLIT

Una vez realizado el SPLIT, y de acuerdo con lo indicado por la aerolínea, la nueva orden generada permitirá la gestión de cambios y reemisiones.

Permisos necesarios para que el usuario pueda realizar SPLIT

Para que un usuario pueda realizar la división de reservas, es necesario contar con el permiso correspondiente configurado en el sistema.

Ingresar al módulo “Admin Backend / Usuarios / Lista de Usuarios”

Figura 4

Buscar y seleccionar al usuario al cual se le desea habilitar el permiso.

Desplazarse hasta la parte final del perfil del usuario hasta ubicar la sección “Permisos”.

Dentro de la categoría “administrador”, identificar el permiso denominado “permite hacer SPLIT”, el cual deberá ser activado para que el usuario pueda realizar la división de reservas.

Figura 5

Una vez realizada la configuración “Guardar” los cambios para que aplique correctamente

Configuración para ocultar la opción “dividir pnr” en usuario tipo cliente.

Para los usuarios con perfil tipo cliente, el sistema cuenta con una configuración adicional que permite ocultar el botón de “Dividir PNR”, en caso de que la agencia requiera restringir el uso de esta funcionalidad. Esta configuración se realiza a nivel de entidad mediante un parámetro del sistema, el cual permite controlar la visualización del botón Split para este tipo de usuarios. Para realizar la configuración se deben seguir los siguientes pasos:

Ingresar al módulo "2 – Admin Backend/configuración parámetros/parámetros Activos".

Figura 6

Buscar y seleccionar la entidad a la cual se le desea habilitar el permiso

Configurar el siguiente parámetro:

-PARÁMETRO: OCULTARSPLITCLIENTE

-DESCRIPCIÓN: Ocultar botón de Split PNR para usuarios tipo cliente-VALORES POSIBLES: S (Sí oculta el botón)

N (No oculta el botón)

Figura 7

Una vez realizada la configuración “Guardar” los cambios para que aplique correctamente

En la siguiente imagen se muestra la visualización del perfil de un usuario “tipo cliente” cuando el parámetro “OCULTARSPLITCLIENTE” está activo, por lo que el botón “Dividir PNR” no se encuentra disponible.

Figura 8

En la siguiente imagen se muestra la visualización del perfil de un usuario “tipo cliente” cuando el parámetro “OCULTARSPLITCLIENTE” no está activo, por lo que el botón “Dividir PNR” se encuentra disponible.

Figura 9
